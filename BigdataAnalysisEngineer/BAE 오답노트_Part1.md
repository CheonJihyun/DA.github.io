## CHAPTER 01 빅데이터의 이해

1. DIKW 피라미드

- 데이터: 가공되기 전의 객관적 수치 또는 기호
- 정보: 데이터의 가공 및 처리를 통해 도출된 현상
- 지식: 정보의 구조화를 통해 도출되는 고유의 아이디어
- 지혜: 지식의 축적과 아이디어가 결합된 창의적 산물

2. 구조에 따른 데이터 유형 분류: 정형, 반정형, 비정형

3. 클라우드 컴퓨팅: 데이터 처리 비용을 낮춰 투자 대비 수익률을 높게 만듦(빅데이터 분석의 비용 효율성을 가져옴)

4. 비정형 데이터

- 메타데이터가 정의되지 않음 (데이터 구조에 대한 메타데이터가 내부에 포함되면 반정형 데이터)
- 수집 데이터 각각 데이터 객체로 구분
- Crawling, RSS, Open API 등으로 수집

5. 지혜: 지식의 축적 + 아이디어

6. 빅데이터 관련 직무

- 데이터 사이언티스트: 통계적 지식과 알고리즘에 대한 이해, 시각화 기법 -> 복잡한 비즈니스 모델을 모델링 + 인사이트 도출
- 데이터 아키텍트: 비즈니스 요건을 구현하기 위한 데이터 흐름과 표준, 원칙 등을 규정
- 데이터 엔지니어: 데이터 플랫폼 설계, 데이터 Flow 관리 및 모델 배포 등 데이터 분석 환경을 설계하고 구축
- 데이터 모델러: 데이터 처리를 위한 모델(논리 모델, 물리 모델)의 설계 및 개발

7. 데이터 -> 정보, 지식, 지혜 얻을 수 있음

8. 빅데이터 분석 조직

- 집중 구조: 각 부서별 분석, 전사 분석업무를 별도의 분석 전담조직에서 담당 -> 분석업무의 이중화/이원화 위험
- 기능 구조: 각 부서별 분석, 별도의 분석 조직 X -> 전사적 핵심 분석 어려움, 과거 실적 분석에 국한될 위험
- 분산 구조: 분석 조직 인력들을 현업 부서로 직접 배치, 분석 결과에 대한 신속한 활동 + Best Practice 공유 가능

9. 빅데이터 분석 조직 구조의 설계 특성

- 공식화: 업무의 수행 절차, 수행 방법, 적업 결과 등의 기준을 사전에 설정
- 분업화: 조직의 목표 달성을 위해 업무를 분할하여 수행
- 직무 전문화: 직무 전문성에 따라 생산성이 증대 -> 전문 지식과 경험이 중요
- 통제 범위: 관리자가 효율적/효과적으로 관리할 수 있는 조직의 인원수 고려
- 의사소통 및 조정: 업무 수행 시 의사소통은 수직적인 활동(업무의 지시, 보고, 피드백 등)과 수평적인 활동(문제 해결을 위한 협업 등)으로 구분

10. 빅데이터의 3V: Volume, Variety, Velocity

11. 데이터 사이언티스트의 역량

- Hard Skill: IT 기술, 분석 기술 등
- Soft Skill: 비즈니스 분석, 의사 전달 능력 등

12. 빅데이터의 활용 및 가치

- 현실 세계를 기반으로 한 패턴 분석, 미래 예측 -> 비즈니스 불확실성 제거
- 새로운 가치 창출, 안정적인 기업 활동 추구
- 다양한 분야 모니터링 -> 위험 징후 사전 탐지
- 다양한 분야와 융합 가능, 방대한 데이터를 활용한 혁신적 가치 창출

13. 빅데이터 출현 배경

- 데이터 저장 기술 발전 -> 저장 비용 감소
- 비정형 데이터 저장/관리 기술 등장
- 기업 활동으로 축적된 데이터를 활용하여 새로운 가치를 창출하고자 하는 니즈
- 일상생활에서 생산되는 데이터의 양이 급격하게 증가

14. 균형 성과표(BSC; Balanced Score Card)의 네 가지 관점: 재무, 고객, 내부 프로세스, 학습/성장

15. 빅데이터가 주는 가치: 미래 대응 가능, 환경 탐색, 상황 분석

16. 스스로 학습하는 기술(i.e. 딥러닝)은 비지도 학습 기법

17. 빅데이터 조직 구조 설계 고려사항: 공식화, 분업화, 직무 전문화, 통제 범위, 의사소통 및 조정

18. 직무별 역량 모델링 개발 단계
    1. 조적의 비전/성과 목표/CSF 검토
    2. 행동 특성 도출
    3. 직무별 역량 도출
    4. 역량 모델 확정

19. 조직성과 평가 단계
    1.  목표 설정
    2.  모니터링
    3.  목표 조정
    4.  평가 시행
    5.  결과 피드백

20. 데이터 사이언티스트의 인문학 열풍

- 컨버전스 -> 디버전스 패러다임 변화
- 제품 생산 -> 서비스로의 변화
- 생산 -> 시장 창조로의 변화

21. 개인정보 비식별화 기술

- 가명처리: 개인 식별이 가능한 데이터를 다른 값으로 대체하는 기법
- 총계처리: 개인 정보에 통곗값을 적용하여 개인을 특정할 수 없게 하는 기법
- 데이터 삭제: 특정 데이터 값을 삭제하는 기법
- 데이터 범주화: 식별값을 해당 그룹의 대푯값이나 구간값으로 변환하는 기법
- 데이터 마스킹: 식별값의 전체 또는 부분을 대체값으로 변환하는 기법

22. 분산구조

- 분석 조직 인력 -> 현업 부서에 직접 배치
- 분석 결과에 따른 신속한 활동, Best Practice 공유 가능
- 각 부서의 분석 업무와 역할 분담을 명확히 해야

23. 산업별 빅데이터 분석 응용

- 제조: 제품 생산 불량률 개선, 수요 예측을 통한 판매 계획 향상
- 쇼핑: 구매 분석을 통한 소비 예측
- 물류: 물류 관리 최적화
- 의료: 헬스케어 서비스를 통한 의료 복지 향상
- 에너지: 트레이딩, 공급/수요 예측
- 정보: 사기탐지, 사례관리, 범죄방지, 수익 최적화

24. 빅데이터를 통한 경영 혁신 발전 단계
    1. 생산성 향상
    2. 발견에 의한 문제 해결
    3. 의사결정 향상
    4. 새로운 가치와 비즈니스 창출

25. 데이터 사이언스와 통계학은 데이터를 분석하는 것은 유사하짐만 다루는 유형과 범위가 다름

26. 분석 기회 발굴

- 역량의 재해석 관점: 보유 역량 뿐 아니라 활용 가능한 역량을 포함하여 분석 기회 탐색
- 거시적 관점: 영향을 비치는 사회/경제적 요인을 나누어서 기회 발굴
- 시장의 니즈 탐색 관점: 직접 고객 외 고객과 접촉하는 채널, 고객 의사결정에 영향을 미치는 인자들의 관점으로 분석 기회 탐색
- 경쟁자 확대 관점: 현재 비즈니스 영역의 직접 경재사 외 대체재, 신규 진입자 등으로 관점을 확대하여 탐색

27. 머신러닝에서 사람이 개입하던 작업 -> 딥 러닝 알고리즘을 통해 컴퓨터가 스스로 분석하고 문제 해결

28. 개인정보 비식별 조치 방법: 가명 처리, 총계 처리, 데이터 삭제, 데이터 범주화, 데이터 마스킹

29. Hard Skill(분석 기술) vs Soft Skill(비즈니스 통찰력, 커뮤니케이션 능력, 협업 능력)

30. 프로젝트 위험 대응 방법: 회피 , 전가, 완화, 수용

31. 빅데이터 조직 구조: 집중 구조, 기능 구조, 분산 구조

## CHAPTER 02 데이터 분석 계획

- 데이터 수집 전에 데이터 유형에 대한 분석 선행되어야
- 데이터 분석 내재화를 위한 변화관리 방안 고려해야
- 분석업무에서 발생 가능한 위험 요소에 대한 대응 계획 수립 해야

- 데이터 수집
- 데이터 저장
- 데이터 처리 기술과 분석
- 분산 데이터베이스: HBase, Cassandra
- 리소스 관리

34. Hard Skill(빅데이터 지식, 분석 기술) vs Soft Skill(통찰력 있는 분석, 의사 전달 능력, 다분야 간 협력)

35. KDD 문석 방법론

- 통계적 패턴이나 지식을 찾기 위해 정리된 데이터 마이닝 프로세스
- 데이터 선택 -> 데이터 전처리 -> 데이터 변환 -> 데이터 마이닝 -> 해석과 평가

36. 데이터: 가공되기 전의 객관적 수치나 사실 -> 정보: 데이터의 가공 및 처리를 통해 도출된 현상

37. 비식별 조치 단계 (개인정보 비식별 조치 가이드라인)

    1. 사전검토
    2. 비식별 조치
    3. 적정성 평가
    4. 사후관리

41. 빅데이터 분석 방법론 계층: 단계, 태스크, 스텝 등

55.  데이터 저장관리 단계
    1. 데이터 전/후 처리
    2. 데이터 저장
    3. 데이터 보안 관리
    4. 데이터 품질관리

56.  명목척도: 임의의 범주로 분류 후 기호/숫자를 부여

57.  데이터 변환 기법

- 특정 구간으로 값을 스케일링
- 주어진 데이터를 토대로 새로운 속성을 생성
- 데이터의 노이즈를 구간화, 군집화 등으로 제거하여 데이터 집합을 매끄럽게 하는 기법
- 다양한 차원으로 데이터를 요약하는 기법

58. Sqoop: 커넥터를 사용하여 하둡과 관계형 데이터베이스의 데이터 전송을 지원하는 수집 도구

59. SCM(Supply Chain Management, 공급망관리): 원재료 수급, 제품 생산, 유통 등의 공급망 활동을 최적화해서 수요에 맞는 제품을 제공할 수 있도록 관리하는 시스템

60. 분석 문제 정의 하향식 접근법

- 문제 탐색 단계

61. 전사적자원관리(ERP; Enterprise Resource Planning) 시스템: 회사의 모든 정보와 재무, 공급망, 제조, 인적자원 등의 비즈니스 프로세스를 통합 관리하며 고객 및 주문정보까지 전사적인 비즈니스를 통합적으로 관리하는 시스템
